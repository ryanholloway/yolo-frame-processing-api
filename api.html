YOLO Object Detection Flask API

Real-time object detection API with YOLO models. Supports live camera and simulation mode for testing.

Endpoints

GET /frame

Returns the latest annotated frame as a JPEG image with detection bounding boxes. If capture hasn't started, returns a message image.

Response: JPEG image

GET /detections

Returns JSON array of the latest detected objects with class names and confidence scores. If capture hasn't started, returns a message.

Response: JSON array, e.g., [{"class_name": "AC", "confidence": 0.95}]

GET /unprocessed_frame

Returns the latest unprocessed frame as a JPEG image without annotations. If capture hasn't started, returns a message image.

Response: JPEG image

GET /model

Returns the current YOLO model name and available models.

Response: JSON, e.g., {"current_model": "yolo11n", "available_models": ["yolo11n"]}

POST /model

Switches the YOLO model used for detection.

Request Body: JSON, e.g., {"model_name": "yolo11n"}

Response: JSON success message or error

GET /health

Returns the health status of the API including model load status and simulation mode.

Response: JSON, e.g., {"status": "healthy", "simulation_mode": false, "model_loaded": true, "model_name": "yolo11n"}

POST /detect

Detects objects in an uploaded image file and returns detections.

Request: multipart/form-data with "image" file field

Response: JSON array, e.g., [{"class_name": "AC", "confidence": 0.95}]

POST /start_capture

Starts the frame capture and detection process.

Response: JSON status message

POST /stop_capture

Stops the frame capture and detection process.

Response: JSON status message

POST /logs

Logs a message to the in-memory log buffer.

Request Body: JSON, e.g., {"level": "INFO", "message": "Something happened", "context": "optional"}

Valid levels: ERROR, WARNING, INFO, DETECTION, DECISION

Response: JSON status message

GET /logs

Returns all logs organized by log level.

Response: JSON object with logs grouped by level (ERROR, WARNING, INFO, DETECTION, DECISION)

GET /logs/<level>

Returns logs for a specific level.

Example: /logs/detection, /logs/error

Response: JSON with logs for the specified level and count

POST /logs/clear

Clears all in-memory logs.

Response: JSON status message

Configuration

The API runs in simulation mode by default for testing. Set SIMULATION_MODE = False in app.py for live camera mode on Raspberry Pi. Capture and detection do not start automatically; use /start_capture to begin.

Models and configurations are defined in config.py.

Usage Examples

# Check API health
curl http://localhost:7926/health

# Get detections from camera stream
curl http://localhost:7926/detections

# Detect objects in uploaded image
curl -X POST -F "image=@image.jpg" http://localhost:7926/detect

# Get annotated frame
curl -o frame.jpg http://localhost:7926/frame

# Switch model
curl -X POST http://localhost:7926/model \
  -H "Content-Type: application/json" \
  -d '{"model_name": "yolo11n"}'

# Start capture
curl -X POST http://localhost:7926/start_capture

# Stop capture
curl -X POST http://localhost:7926/stop_capture
